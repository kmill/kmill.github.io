Universality and quines

Quines, named by Douglas Hofstadter in honor of Willard Van Orman Quine, are self-replicating programs, programs that output their own source code. It’s actually just a consequence of Turing completeness of a programming language that it’s possible to construct a quine. For example, here’s a quine in a sort of classic structure in Python:[1] 

a = ['print "a =", a', 'for s in a: print s']
print "a =", a
for s in a: print s

In this note, I’m going to show how to make a lambda calculus interpreter in lambda calculus (a “universal machine”), how to use it to prove a kind of fixed-point theorem for the lambda calculus, and then how to get a quine from it.

1. The lambda calculus

Alonzo Church came up with a fairly nice and simple calculus that can be used as a basis for both computation and mathematics. In his version, there are three types of symbolic expressions:

  • xx if xx is a symbol (variable), where we presume we have an infinite set of symbols to draw from.
  • (x↦M)(x↦M) if xx is a symbol and MM is a lambda calculus term (abstraction).[2]
  • (M N)(M N) if MM and NN are lambda calculus terms (application).
This sort of abstractly models the idea of functions and applying functions to values, but with the oddity that everything is a function. The “semantics” are defined by an equivalence relation on lambda calculus terms. The definitions depend quite a lot on the concept of a free variable, which can be inductively defined as such:
  • free(x)={x}free⁡(x)={x},
  • free(x↦M)=free(M)−{x}free⁡(x↦M)=free⁡(M)−{x}, and
  • free(M N)=free(M)∪free(N)free⁡(M N)=free⁡(M)∪free⁡(N).
The xx in x↦Mx↦M is called a bound variable. We also need substitution M[x→N]M[x→N] that replaces all free instances of xx with NN:
  • y[x→N]={Nif x=yyotherwisey[x→N]={Nif x=yyotherwise
  • (y↦M)[x→N]={(y↦M)if x=y(y↦M[x→N])otherwise(y↦M)[x→N]={(y↦M)if x=y(y↦M[x→N])otherwise where y∉free(N)y∉free⁡(N)
  • (M1 M2)[x→N]=(M1[x→N] M2[x→N])(M1 M2)[x→N]=(M1[x→N] M2[x→N]).
This is a partial function because of the constraint in the second rule, but up to the following equivalence relations one may think of it as being a function.
αα-renaming:
If (x↦M)(x↦M) is a lambda term and y∉free(M)y∉free⁡(M), (x↦M)≡(y↦M[x→y])(x↦M)≡(y↦M[x→y]).
ββ-reduction:
If ((x↦M) N)((x↦M) N) is a lambda term, then ((x↦M) N)≡M[x→N]((x↦M) N)≡M[x→N].
ηη-conversion:
If x∉free(M)x∉free⁡(M), then M≡(x↦(M x))M≡(x↦(M x)).

There’s not really a sense in which a lambda calculus term is evaluated per se, but there are various normal forms one could try to put a particular lambda calculus term in (as functional programming languages do). But, reduction strategies have divergent terms, which are lambda calculus terms that do not stabilize while applying the reduction strategy. For example, Ω=((x↦(x x)) (x↦(x x)))Ω=((x↦(x x)) (x↦(x x))) is generally divergent.

As a shorthand, we will take application to be left-associative, meaning (L M N)(L M N) is ((L M) N)((L M) N), and abstraction to be right-associative, meaning (x↦y↦M)(x↦y↦M) is (x↦(y↦M))(x↦(y↦M)). One usually thinks of (x↦y↦M)(x↦y↦M) as being a two-argument function.[3] We will also sometimes not write outermost parentheses, and we will use the rule that x↦M Nx↦M N means (x↦(M N))(x↦(M N)).

1.1. de Bruijn indexes

Symbolic variables are an annoying complexity for lambda calculus because of all the issues with free variables in αα-renaming and substitution. One way around this is to use the natural numbers N={0,1,2,…}N={0,1,2,…} for variables, where the number represents how many abstractions out that variable is bound from.

Lambda calculus terms when using de Bruijn indexes are

  • nn if n∈Nn∈N (variable),
  • (λM)(λM) if MM is a term (abstraction), and
  • (M N)(M N) if MM and NN are terms (application).

Then, the equivalence relation no longer needs αα-renaming. Substitution is

  • m[n→N]={Nif m=nmotherwisem[n→N]={Nif m=nmotherwise
  • (λM)[n→N]=(λM[n+1→N+])(λM)[n→N]=(λM[n+1→N+])
  • (M1 M2)[n→N]=(M1[n→N] M2[n→N])(M1 M2)[n→N]=(M1[n→N] M2[n→N]).
The N+N+ denotes incrementing each unbound variable appearing in NN (this can be thought of as the simultaneous application of the substitutions [n→n+1][n→n+1] for all n∈Nn∈N).

The equivalence relation is generated by two rules.

ββ-reduction:
((λM) N)≡M[0→N+]((λM) N)≡M[0→N+].
ηη-conversion:
M≡(λ(M+ 0))M≡(λ(M+ 0)).

For lambda calculus terms with no free variables, it is a purely mechanical process going back and forth between de Bruijn indexes. One way to start is to αα-rename all the bound variables so that they come from some alphabet x0,x1,x2,…x0,x1,x2,… so that each nested abstraction uses the next variable in the list, and then the index is d−id−i where ii is from xixi and dd is the abstraction depth. Terms with free variables can be dealt with by imagining a that those variables have been bound in some specified order.

I sort of like the following variation of de Bruijn indexes, where variables are written in unary rather than from the infinite alphabet NN.

  • zz is a term (representing zero in NN).
  • SMSM is a term if MM is a term (the successor, representing n+1n+1 in NN but extended to all terms).
  • (λM)(λM) is a term if MM is a term.
  • (M N)(M N) is a term if MM and NN are terms.
The substitution rules M[n→N]M[n→N] with n∈Nn∈N are
  • z[n→N]={Nif n=0zotherwisez[n→N]={Nif n=0zotherwise
  • (SM)[0→N]=SM(SM)[0→N]=SM
  • (SM)[n+1→N]=SM[n→N](SM)[n+1→N]=SM[n→N] (Errata: [4])
  • (λM)[n→N]=(λM[n+1→SN])(λM)[n→N]=(λM[n+1→SN])
  • (M1 M2)[n→N]=(M1[n→N] M2[n→N])(M1 M2)[n→N]=(M1[n→N] M2[n→N]).
Then, while ββ-reduction is the same, ηη-conversion is simply M≡(λ(SM z))M≡(λ(SM z)). However, this is at the cost of needing an SS-reduction rule, the details of which I’ll spare you because they’re easy to figure out if you need them.

2. Algebraic data types

Pretty much every data type that you want can be represented as some expression of products and disjoint unions of basic data types, possibly with some recursion. For example, if ⊤⊤ represents a singleton type, the the type LL(X)LL(X) for a linked list of things of type XX is recursively defined by LL(X)=⊤⊔(X×LL(X))LL(X)=⊤⊔(X×LL(X)). These sorts of algebraic expressions to represent types are ADTs (algebraic data types).

In lambda calculus, a value (x,y)(x,y) of a product type X×YX×Y can be represented by the expression f↦f x yf↦f x y, where the projector functions are πX=p↦p (x↦y↦x)πX=p↦p (x↦y↦x) and πY=p↦p (x↦y↦y)πY=p↦p (x↦y↦y). The constructor function is x↦y↦f↦f x yx↦y↦f↦f x y. The idea is that a value is something that provides its component values to some supplied function.

A disjoint union X⊔YX⊔Y, on the other hand, is described by its inclusion functions: ιX=x↦(f↦g↦f x)ιX=x↦(f↦g↦f x) and ιY=y↦(f↦g↦g y)ιY=y↦(f↦g↦g y), which is a sort of dual to the projectors for the product type. The idea here is that a value is something that selects a function to handle which of the types the value is from.

A famous disjoint union is the two-valued boolean algebra, of two singleton types. In an optimized representation that doesn’t use the values from the singleton types,

T=x↦y↦xF=x↦y↦yT=x↦y↦xF=x↦y↦y so if bb is a boolean, an if statement looks like (b x y)={xif b=Tyif b=F(b x y)={xif b=Tyif b=F.[5]

In the case of products and disjoint unions of more than two types, there is an obvious optimization that one can apply. For example, for X×Y×ZX×Y×Z we can use x↦y↦z↦p x y zx↦y↦z↦p x y z as a constructor.

2.1. Representing lambda calculus terms

The point of introducing ADTs is that we can use them to represent lambda calculus terms within the lambda calculus itself. While it is true that technically a lambda calculus term is a string, we will be representing it in a tree form. In particular, we will represent the variation on lambda calculus terms from the end of the de Bruijn indexes section.

Let LL denote the type for a lambda calculus term, for which we give the following recursive ADT description

L=⊤⊔L⊔L⊔(L×L)L=⊤⊔L⊔L⊔(L×L) where the four terms correspond to zz, successor, abstraction, and application. In particular, we’ll use these constructors:

  • zero=z↦s↦l↦a↦zzero=z↦s↦l↦a↦z
  • succ(e)=z↦s↦l↦a↦s esucc⁡(e)=z↦s↦l↦a↦s e
  • abs(m)=z↦s↦l↦a↦l mabs⁡(m)=z↦s↦l↦a↦l m
  • apply(m,n)=z↦s↦l↦a↦a m napply⁡(m,n)=z↦s↦l↦a↦a m n

As an example, the expression for TT can be expressed in this type as abs(abs(succ(zero)))abs⁡(abs⁡(succ⁡(zero))).

If we have an expression pp from LL, we can decompose it and convert it into a lambda calculus term. Write I(p)I(p) for the corresponding term, the II being for “interpretation.” This II is a function outside the lambda calculus, but we will show the lambda calculus is powerful enough to implement it.

3. An interpreter

In this section we’ll derive a term uu such that when pp is an expression representing a lambda calculus term from the type LL, then u p≡I(p)u p≡I(p). This interpreter is the lambda calculus equivalent of a universal machine, because it can take descriptions of lambda terms and convert them into actual lambda terms, compared to descriptions of Turing machines being run as actual Turing machines.

At a high level, the term looks like the following, in a pseudo-Haskell notation. We make the decision that free variables in pp evaluate to a diverging term. The env variable is a linked list of lambda calculus terms (actual terms, not representations of them). 

u p := eval p empty
eval p env := case p of
    zero -> head env
    succ(e) -> eval e (tail env)
    abs(m) -> (x |-> eval m (push x env))
    app(m1, m2) -> (eval m1 env) (eval m2 env)
empty := nil |-> cons |-> nil
push x list := nil |-> cons |-> cons x list
head list := list fail (x |-> sublist |-> x)
tail list := list empty (x |-> sublist |-> sublist)
fail := fail
The way this works is that the eval function keeps track of an environment of all the substitutions from ββ-reduction. The interpretation of abs(m)abs⁡(m) is as an actual lambda abstraction that pushes the value of the bound x to the zeroth entry of the environment list. Furthermore, empty is the empty linked list, push is the constructor, and head and tail are convenience destructors, where head fails in the case the environment is an empty list.

I’m afraid I’m going to be lazy and not explicitly prove that u p≡I(p)u p≡I(p), but it is a matter of checking the equivalence relations (and the substitution rule). Note that we do not care about evaluation strategies because the point is just that u pu p is a lambda term in the right equivalence class.

Now, how do we deal with recursion when the lambda calculus does not seem to provide it? The YY combinator is a way to let a term refer to itself, and its defining relation is that Y f≡f (Y f)Y f≡f (Y f). Thus, if a function needs to refer to itself, like in the factorial function 

factorial n := if n <= 1
    then 1
    else n * factorial (n - 1)
we can transform it like so 
factorial' fact n := if n <= 1
    then 1
    else n * fact (n - 1)
factorial := Y factorial'
where none of the definitions are self-referential. The trick is that 
factorial n = Y factorial' n
            = factorial' (Y factorial') n
            = factorial' factorial n
where the equality signs actually mean equivalence, and hence factorial’ is able to refer to something that is equivalent to the factorial expression itself. This YY combinator can actually be given as a lambda calculus term: Y=f↦(x↦f (x x)) (x↦f (x x))Y=f↦(x↦f (x x)) (x↦f (x x)) The defining relation is not too difficult to check.

Let’s look at the fail function, which we could define as 

fail := Y fail'
fail' f := f
hence fail is Y (x↦x)≡(x↦(x x)) (x↦(x x))=ΩY (x↦x)≡(x↦(x x)) (x↦(x x))=Ω, the divergent term from before.

Let’s rewrite the definition for u to be non-self-referential, while also substituting in some of the definitions for linked lists. 

u p := (Y eval') p (n |-> c |-> n)
eval' eval p env := case p of
  zero -> env omega (x |-> t |-> x)
  succ(e) -> eval e (env env (x |-> t |-> t))
  abs(m) -> (x |-> eval m (n |-> c |-> c x env))
  app(m1, m2) -> (eval m1 env) (eval m2 env)
omega := (x |-> x x) (x |-> x x)
Y f = (x |-> f (x x)) (x |-> f (x x))
For succ(e), there is a trick that if env is empty then by passing env as the first argument the expression is equivalent to the empty list in that case.

Now, to write this in an impenetrable way as a single lambda term: 

u := expr |->
  ((f |-> (x |-> f (x x)) (x |-> f (x x)))
    (eval |-> p |-> env |->
     p (env ((x |-> x x) (x |-> x x)) (x |-> t |-> x))
       (e |-> eval e (env env (x |-> t |-> t)))
       (m |-> x |-> eval m (n |-> c |-> c x env))
       (m1 |-> m2 |-> (eval m1 env) (eval m2 env)))
  ) expr (n |-> c |-> n)

Or, if that was too clear, using de Bruijn indexes:

u=λ((λ(λ1(00))(λ1(00)))(λλλ1(0((λ00)(λ00))(λλ1))(λ30(11(λλ0)))(λλ41(λλ024))(λλ(412)(402))))0(λλ1)u=λ((λ(λ1(00))(λ1(00)))(λλλ1(0((λ00)(λ00))(λλ1))(λ30(11(λλ0)))(λλ41(λλ024))(λλ(412)(402))))0(λλ1) The point of this last exercise was just to make it obvious that uu can be represented as an expression UU. If everything worked out, then in fact u U≡uu U≡u.

If we do not care about the behavior for free variables, we could shorten this term by replacing ΩΩ with something random, like 00 for instance, which would have the effect of making free variables evaluate to the empty list.

It’s also possible to write a universal interpreter that enforces a particular reduction strategy, for example the Krivine machine[6], with a short implementation by Scott Kovach.

4. A fixed-point theorem

Consider a lambda term ff with the property that for every lambda expression pp from LL, f pf p is equivalent to something in LL. That is, ff can be thought of as an actual function L→LL→L. We will show that then ff has a fixed point with respect to uu, namely that there is some expression qq from LL such that u (f q)≡u qu (f q)≡u q. (The argument is adapted from the one presented here.[7])

A key operator in the construction of qq is quote:L→Lquote:L→L which takes a lambda expression and produces a lambda expression that when evaluated is a term that is equivalent to that expression. It’s much less complicated to say in symbols: u quote(p)≡pu quote⁡(p)≡p. We can (tediously) define this operator inductively.[8]

  • quote(zero)=abs(abs(abs(abs(succ(succ(succ(zero)))))))quote⁡(zero)=abs⁡(abs⁡(abs⁡(abs⁡(succ⁡(succ⁡(succ⁡(zero)))))))
  • quote(succ(e))=abs(abs(abs(abs(apply(succ(succ(zero)),quote(e))))))quote⁡(succ⁡(e))=abs⁡(abs⁡(abs⁡(abs⁡(apply⁡(succ⁡(succ⁡(zero)),quote⁡(e))))))
  • quote(abs(m))=abs(abs(abs(abs(apply(succ(zero),quote(m))))))quote⁡(abs⁡(m))=abs⁡(abs⁡(abs⁡(abs⁡(apply⁡(succ⁡(zero),quote⁡(m))))))
  • quote(apply(m,n))=abs(abs(abs(abs(apply(apply(zero,quote(m)),quote(n))))))quote⁡(apply⁡(m,n))=abs⁡(abs⁡(abs⁡(abs⁡(apply⁡(apply⁡(zero,quote⁡(m)),quote⁡(n))))))
Hopefully it’s clear how to write down a lambda term that actually implements quotequote, but it’s similar to the way the case expression in uu was implemented.

Let s:L→Ls:L→L be defined by s p=apply(p,quote(p))s p=apply⁡(p,quote⁡(p)), which creates an expression that represents supplying the representation of a lambda term to itself as an argument.

Let mm be the expression for p↦u (f (s p))p↦u (f (s p)), and let q=s mq=s m.

u (f q)≡u (f (s m))≡(p↦u (f (s p))) m≡(u m) m≡u apply(m,quote(m))≡u (s m)≡u q.u (f q)≡u (f (s m))≡(p↦u (f (s p))) m≡(u m) m≡u apply⁡(m,quote⁡(m))≡u (s m)≡u q. Hence, qq is indeed a fixed point of ff with respect to uu.

5. Quines

A quine is an expression qq in LL such that u q≡qu q≡q.

Quines are the result of applying this fixed-point theorem to the quotequote operator. By that theorem, there is some qq in LL such that u quote(q)≡u qu quote⁡(q)≡u q, but since u quote(q)≡qu quote⁡(q)≡q, we have obtained a quine.

What does this quine look like? We had that mm was the expression for p↦u quote(apply(p,quote(p))p↦u quote⁡(apply⁡(p,quote⁡(p)), but this simplifies to p↦apply(p,quote(p))p↦apply⁡(p,quote⁡(p)), so we may let m=abs(apply(apply(A,zero),apply(Q,zero)))m=abs⁡(apply⁡(apply⁡(A,zero),apply⁡(Q,zero))), where AA is the lambda expression for applyapply and QQ is a lambda expression for quotequote. Then, qq is apply(m,quote(m))apply⁡(m,quote⁡(m)), which expanded a bit is

q=apply(abs(apply(apply(A,zero),apply(Q,zero))),quote(abs(apply(apply(A,zero),apply(Q,zero)))))q=apply⁡(abs⁡(apply⁡(apply⁡(A,zero),apply⁡(Q,zero))),quote⁡(abs⁡(apply⁡(apply⁡(A,zero),apply⁡(Q,zero))))) It doesn’t seem like it would be too illuminating to expand this out anymore, but it is interesting that uu was eliminated so easily, suggesting the fixed-point theorem was tremendously overpowered for the purpose of deriving a quine.

Anyway, notice that the way qq works is that there is a code part and a data part, and that QQ is sort of like a print function. There’s sort of a parallel between the structure of qq and the Python program from the beginning.

6. In set theory

As I understand it, one of the ways to study ZFC set theory is by model theory, where a model is a set MM along with a relation (∈)⊂M×M(∈)⊂M×M that satisfies all of the axioms. Propositions of first-order logic can themselves be represented as sets (like how lambda terms can be represented in LL as lambda calculus terms), and there is a truth term that evaluates the truth of a particular proposition against a particular model.

Unfortunately there is no way to prove in ZFC that there is a model of ZFC (that’s Gödel’s second incompleteness theorem).

As an application of the idea in the fixed-point theorem, let’s show that there’s no absolute notion of Truth in ZFC. If you want, that there’s no truth term τ(p)τ(p) that evaluates whether or not pp is a true proposition in the current model.

The idea is that we can use proposition-level functions defined on set representations of propositions. If f:P→Pf:P→P is a proposition-level function on the set of propositions PP (i.e., ∀x∈P1,∃!y,y∈P1∧f(x,y)∀x∈P1,∃!y,y∈P1∧f(x,y), where we will write f(x)f(x) for this yy and leave it up to the reader to transform the first-order statement appropriately), then we can show that there is some statement q∈Pq∈P such that τ(f(q))⇔τ(q)τ(f(q))⇔τ(q). Let P1P1 be the set of set representations of propositions with one free variable (predicates). We can define proposition-level function s:P1→Ps:P1→P that feeds a predicate the representation of the predicate. Then, we can define a predicate F(p)=τ(f(s(p)))F(p)=τ(f(s(p))) and thus a proposition q=s(F)∈Pq=s(F)∈P, the required fixed point.

But, what if ff is the function p↦¬pp↦¬p that negates a proposition? Then we have a proposition qq such that τ(¬q)⇔τ(q)τ(¬q)⇔τ(q), so ττ can’t possibly be a truth term in actuality. My intuition for what is going wrong is that ττ seems like it would need to have an environment to keep track of all of the bound variables, but ZFC prevents you from having sets with arbitrary elements: you always have to specify a domain. Well, the domain would be the model, but you don’t have a way to gain direct access to the current model from within the model. If you did, you’d have Russell’s paradox.

A kind of fun modification to this is to say “well, ok, fine, there’s no τ(p)τ(p) predicate, but we can still have have a universal truth predicate T(p)=∀M,model(M)⇒τ(p,M)T(p)=∀M,model⁡(M)⇒τ(p,M) that checks the truth across all models, since you can write down a predicate to check whether a set is a model of set theory, so what happens now?”[9] Here’s what happens: the kind of argument above gives us a proposition qq such that T(q)⇔T(¬q)T(q)⇔T(¬q). Gödel’s completeness theorem says that T(p)T(p) is equivalent to pp being provable, and so qq being provable is equivalent to ¬q¬q being provable. So, if the theory is consistent, neither qq nor ¬q¬q are provable (the law of the excluded middle does not apply to provability). The theory is incomplete!
